# MVP 架构与实施路线

## 推荐工作流

```text
用户问题
  -> 意图与基金代码解析
  -> 数据路由（缓存优先，AKShare/官方披露补充）
  -> 数据质量检查（日期、缺失、重复、口径）
  -> 确定性指标计算
  -> 同类/基准/持仓分析
  -> 证据检索与引用
  -> 报告生成
  -> 合规与一致性审校
  -> 用户结果
```

语言模型负责意图理解、工具选择、解释和报告组织；净值处理、指标计算、排名与图表数据由 Python 工具完成。

## 最小模块

```text
funding/
  src/
    data/          # 数据适配器、缓存、统一 schema
    analytics/     # 收益、风险、回撤、基准和同类比较
    research/      # 文档检索、证据与引用
    agent/         # 工具注册、工作流和输出审校
    reporting/     # 报告模板与图表
  skills/          # 项目内领域 SKILL.md
  tests/
    fixtures/      # 固定样本，避免测试依赖实时网络
    golden/        # 指标与报告金标准
  docs/
```

## 统一数据对象

首期至少定义：

- `FundIdentity`：代码、名称、份额类别、基金类型、成立日、管理人与托管人。
- `NavPoint`：净值日期、单位净值、累计净值、复权标记、来源。
- `HoldingSnapshot`：报告期、披露日、资产类别、证券代码、权重、来源。
- `BenchmarkSpec`：基准名称、组成规则、行情序列来源。
- `MetricResult`：指标名、值、区间、频率、参数、数据质量标记。
- `Evidence`：事实文本、来源链接、文档日期、页码/段落、抓取时间。

## 三阶段路线

### 阶段 1：单基金分析

- 基本信息与净值曲线。
- 收益、波动、最大回撤、夏普和卡玛。
- 与一个明确基准比较。
- 输出带数据日期与来源的 Markdown/HTML 报告。
- 固定样本单元测试和接口失败降级。

### 阶段 2：同类与持仓

- 基金分类和可比基金池。
- 同类分位、滚动指标和经理任期切片。
- 重仓重合度、行业暴露、集中度和披露滞后说明。
- PDF 定期报告证据提取。

### 阶段 3：组合与持续监控

- 多基金组合归因、相关性和压力测试。
- 组合优化与约束解释。
- 数据异常、公告和指标变化监控。
- 研究报告版本管理与人工审批。

## 首期验收标准

- 任意支持的基金代码都能明确返回“成功、无数据或不支持”，不能静默失败。
- 同一份固定净值数据重复运行得到完全相同的指标。
- 报告中的每个关键数值能追溯到输入序列和计算参数。
- 外部事实带来源与日期；过期持仓必须显示报告期。
- 数据冲突、缺失和接口失败不会转化为确定性投资结论。
- 输出包含研究用途声明，并经过合规检查步骤。
