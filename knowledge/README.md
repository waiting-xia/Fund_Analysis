# 金融理论知识库维护说明

这里存放相对稳定的金融、经济和基金研究理论。实时净值、行情、新闻、公告、基金当期持仓和模型生成结论不应写入本目录，它们应继续从真实数据接口按时间获取。

## 文档格式

使用 UTF-8 Markdown 或纯文本。推荐的 Markdown 头部如下：

```markdown
---
title: 文档标题
category: 风险管理
tags: 最大回撤,波动率,相关性
version: 1
---

# 章节名称

正文内容……
```

- `title` 用于模型引用，保持明确且稳定。
- `category` 用于限定检索范围。
- `tags` 使用英文逗号分隔，填写常见金融术语及同义表达。
- 标题会成为检索片段的章节名；单个章节宜聚焦一个问题。
- 结论要写明适用条件、限制和常见误用，避免无条件的方向判断。

## 索引与检索

在 `web/` 目录执行：

```powershell
npm.cmd run rag:build
npm.cmd run rag:status
conda run --no-capture-output -n Python_310 python mcp/rag_admin.py search "最大回撤与波动率的区别" --top-k 5
```

索引使用 SQLite 保存。检索组合了中文字符/词组哈希向量和关键词覆盖率，完全在本机运行。文档修改后无需手工删除数据库：检索前会比较内容哈希并自动重建。

## 质量检查

新增内容前确认：

1. 内容是一般理论或可长期复用的研究方法，而非某天的市场判断。
2. 定义、指标口径和适用边界完整。
3. 没有复制受版权保护的长篇材料；优先用原创总结并记录允许使用的来源。
4. 不包含 API Key、客户持仓、交易记录等敏感信息。
5. 用典型问题执行一次检索，确认目标章节能进入前五条结果。
